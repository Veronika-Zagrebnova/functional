# Отчет по решению задач на алгоритмы

**Динамическое программирование** - это метод решения сложных задач путем разбиения их на более простые подзадачи. Основная идея заключается в том, что мы решаем каждую подзадачу только один раз и сохраняем результат, чтобы избежать повторных вычислений.

**Ключевые характеристики:**
- **Оптимальная подструктура** - оптимальное решение задачи может быть составлено из оптимальных решений ее подзадач
- **Перекрывающиеся подзадачи** - задача может быть разбина на подзадачи, которые решаются многократно
- **Мемоизация** - сохранение результатов вычислений для повторного использования

**Big O нотация** - это математическая нотация, которая описывает асимптотическое поведение функции. В программировании она используется для описания временной и пространственной сложности алгоритмов.

**Основные классы сложности:**
- **O(1)** - постоянная сложность
- **O(log n)** - логарифмическая сложность
- **O(n)** - линейная сложность
- **O(n log n)** - линейно-логарифмическая сложность
- **O(n²)** - квадратичная сложность
- **O(2ⁿ)** - экспоненциальная сложность
- 
**Применение:**
- Анализ эффективности алгоритмов
- Сравнение производительности разных подходов
- Прогнозирование времени выполнения на больших данных

**Мемоизация** - это техника оптимизации, при которой результаты дорогостоящих вызовов функций сохраняются и возвращаются из кэша при повторных вызовах с теми же параметрами.

**Преимущества:**
- Значительное ускорение рекурсивных алгоритмов
- Устранение избыточных вычислений
- Сохранение элегантности рекурсивного подхода
- 
**Реализация:**
- Использование словаря для хранения вычисленных значений
- Проверка наличия результата перед вычислением
- Сохранение результата после вычисления

### Задача 1: Подъем по лестнице (Climbing Stairs)
**Суть задачи:** Необходимо подняться по лестнице из n ступенек. За один шаг можно подняться на 1 или 2 ступеньки. Требуется найти количество способов подняться на вершину.

<img width="709" height="692" alt="{41C2FE0B-9A87-4B24-8B7C-970FDC708398}" src="https://github.com/user-attachments/assets/e8435258-3bbd-49a8-a104-488f29771cc2" />

**Принцип работы:**
- Используется подход динамического программирования
- Количество способов для ступеньки n равно сумме способов для ступенек n-1 и n-2
- Временная сложность: O(n), пространственная: O(1)

### Задача 2: Прыжки до конца массива (Jump Game II)
**Суть задачи:** Дан массив, где каждый элемент показывает максимальную длину прыжка из этой позиции. Найти минимальное количество прыжков для достижения последнего элемента.

<img width="555" height="722" alt="{3A897600-F9E8-44F9-B13E-9B4DD6EEBE1F}" src="https://github.com/user-attachments/assets/b161441f-b044-407c-b64f-9651e2ce2f4e" />

**Принцип работы:**
- Жадный алгоритм с отслеживанием текущей достижимой границы
- На каждом шаге обновляется самая дальняя достижимая позиция
- Когда достигается текущая граница, совершается прыжок
- Временная сложность: O(n), пространственная: O(1)

### Задача 3: Строка треугольника Паскаля (Pascal's Triangle II)
**Суть задачи:** Для заданного индекса строки вернуть соответствующую строку треугольника Паскаля.

<img width="563" height="512" alt="{2E9A663B-48FE-439F-86FF-B4034BC5BB24}" src="https://github.com/user-attachments/assets/9c756044-21a7-4f8a-b64e-506a2a66c187" />

**Принцип работы:**
- Используется единственный массив, который обновляется справа налево
- Каждый элемент вычисляется как сумма текущего и предыдущего элемента
- Обратный порядок обхода предотвращает перезапись нужных значений
- Временная сложность: O(n²), пространственная: O(n)

### Задача 4: Максимальная прибыль от одной сделки (Best Time to Buy and Sell Stock)
**Суть задачи:** Найти максимальную прибыль от покупки и продажи акций, если можно совершить только одну сделку.

<img width="560" height="543" alt="{56C6FDB6-500D-46C1-BFD0-757F77977278}" src="https://github.com/user-attachments/assets/89d3b222-e741-4d0a-9c1d-a2462c77c5e3" />

**Принцип работы:**
- Отслеживается минимальная цена и максимальная возможная прибыль
- Для каждой цены вычисляется потенциальная прибыль
- Временная сложность: O(n), пространственная: O(1)

### Задача 5: Максимальная прибыль от многократных сделок (Best Time to Buy and Sell Stock II)
**Суть задачи:** Найти максимальную прибыль от неограниченного количества покупок и продаж акций.

<img width="559" height="517" alt="{894B0C29-AA4B-457D-8BD4-6BBC72838268}" src="https://github.com/user-attachments/assets/2dd60c1a-9147-4d88-98e4-7e68772c680a" />

**Принцип работы:**
- Жадный алгоритм, собирающий прибыль со всех восходящих движений
- Прибыль добавляется каждый раз, когда цена растет
- Временная сложность: O(n), пространственная: O(1)
