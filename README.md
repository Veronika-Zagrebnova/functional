# Отчет по решению задач на алгоритмы

**Динамическое программирование** - это метод решения сложных задач путем разбиения их на более простые подзадачи. Основная идея заключается в том, что мы решаем каждую подзадачу только один раз и сохраняем результат, чтобы избежать повторных вычислений.
**Ключевые характеристики:**
- **Оптимальная подструктура** - оптимальное решение задачи может быть составлено из оптимальных решений ее подзадач
- **Перекрывающиеся подзадачи** - задача может быть разбина на подзадачи, которые решаются многократно
- **Мемоизация** - сохранение результатов вычислений для повторного использования

**Big O нотация** - это математическая нотация, которая описывает асимптотическое поведение функции. В программировании она используется для описания временной и пространственной сложности алгоритмов.
**Основные классы сложности:**
- **O(1)** - постоянная сложность
- **O(log n)** - логарифмическая сложность
- **O(n)** - линейная сложность
- **O(n log n)** - линейно-логарифмическая сложность
- **O(n²)** - квадратичная сложность
- **O(2ⁿ)** - экспоненциальная сложность
**Применение:**
- Анализ эффективности алгоритмов
- Сравнение производительности разных подходов
- Прогнозирование времени выполнения на больших данных

- **Мемоизация** - это техника оптимизации, при которой результаты дорогостоящих вызовов функций сохраняются и возвращаются из кэша при повторных вызовах с теми же параметрами.
**Преимущества:**
- Значительное ускорение рекурсивных алгоритмов
- Устранение избыточных вычислений
- Сохранение элегантности рекурсивного подхода
**Реализация:**
- Использование словаря для хранения вычисленных значений
- Проверка наличия результата перед вычислением
- Сохранение результата после вычисления

### Задача 1: Подъем по лестнице (Climbing Stairs)
**Суть задачи:** Необходимо подняться по лестнице из n ступенек. За один шаг можно подняться на 1 или 2 ступеньки. Требуется найти количество способов подняться на вершину.
